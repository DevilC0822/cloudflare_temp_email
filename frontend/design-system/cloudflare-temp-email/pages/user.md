# 用户页（User）— 设计说明

## 核心目标

- 登录状态明确：顶部仅显示「当前用户」与邮箱（tag），未登录时展示登录卡片。
- 功能分区清晰：地址（管理/创建/绑定）与收件（用户邮件）分离。
- 安全相关（Passkey/退出）独立为「设置」或「安全」模块，避免混杂在列表操作里。

## Tab 建议

保持 4 个 Tab，但缩短文案（中文 2–4 字）：

- 地址管理：地址
- 收件箱：收件
- 用户设置：安全（或 设置）
- 绑定邮箱地址：绑定

## 交互建议

- 支持 URL Query：`?tab=` 对应当前用户 Tab，刷新/分享不丢失位置。
- 表格操作（切换/转移/解绑）保持“危险动作”二次确认；确认文案 1 句讲清楚后果。
