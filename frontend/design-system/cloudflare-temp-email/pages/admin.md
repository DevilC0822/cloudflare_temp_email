# 管理员页（Admin）— 设计说明

## 核心目标

- 强层级：管理功能多，必须靠“分组 + 清晰标题”降低认知负担。
- 默认推荐左侧导航（桌面端）：避免顶部 Tab 过长导致拥挤。
- 退出登录要明确：仅当使用 Admin 密码登录时显示退出入口，并二次确认。

## 功能分组（顶层）

- 快速设置：数据库 / 账号设置 / 用户设置 / Worker 配置
- 账号：账号列表 / 创建账号 / 账号设置 / 发件权限 / IP 黑名单 / AI 提取 / Webhook
- 用户：用户管理 / 用户设置 / OAuth2 / 角色地址配置
- 邮件：邮件 / 未知收件人 / 发件箱 / 发送邮件 / 邮件 Webhook
- Telegram：机器人配置
- 统计：统计面板
- 维护：数据库 / Worker 配置 / 维护工具
- 外观：外观设置
- 管理员：登录方式 + 退出
- 关于：项目与社区入口

## 交互建议

- 支持 URL Query：`?tab=` 记忆顶层位置（避免刷新丢失）。
- 内层 Tabs（bar）保持居中与紧凑，但不要出现过长文案；必要时改短。
